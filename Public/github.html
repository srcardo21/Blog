Repositoio Remoto

Los repositorios remotos son versiones de tu proyecto que están hospedadas en Internet o en cualquier otra red.

1. Para empezar debemos tener ya creado nuestro repositorio local.

2. Una vez creado, vamos a GitHub y creamos un nuevo repositorio.

◉ En caso de que aún no tengamos creado el repositorio local, GitHub nos brinda los comandos necesarios para la creación.

git init
git add README.md
git commit -m "first commit

3 Ya teniendo el repositorio Local y el remoto creado, vamos a nuestra consola y usamos el comando git remote add origin https://github.com/ejemplo.git (La url se genera después de crear el repositorio).

Con este comando sincronizamos el repositorio local y el remoto.

4 Luego ejecutamos el comando git branch -M main, que Renombra la rama actual a "main" y queda apuntando a esta.

5 Por ultimo ejecutamos el comando git push -u origin main para subir los cambios locales de tu rama actual a un repositorio remoto.

Notas

◉ Cuando ejecutemos el comando git push -u origin main nos debería salir una ventana emergente para iniciar sesión en GitHub.

◉ Si tratamos de ejecutar el comando git push -u origin main, sin guardar los cambios recientes del repositorio local nos va a generar un error.

Una vez realizados todos los pasos anteriores, ya tendríamos los repositorios sincronizados y con la información actualizada.
